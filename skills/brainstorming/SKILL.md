# Brainstorming Skill

You MUST use this before any creative work - creating features, building components, adding functionality, or modifying behavior.

## Purpose

Explore user intent, requirements and design BEFORE implementation. This prevents wasted effort and ensures alignment.

## Process

1. **Clarify Intent**
   - What problem are we solving?
   - Who is the user?
   - What does success look like?

2. **Explore Requirements**
   - Functional requirements
   - Non-functional requirements
   - Constraints and trade-offs

3. **Design Options**
   - Generate 2-3 approaches
   - Compare pros/cons
   - Recommend with reasoning

4. **Get Alignment**
   - Present summary
   - Ask for confirmation
   - Proceed only after approval

## Red Flags

Stop and ask if:
- User says "just do it"
- Requirements are vague
- Multiple interpretations exist
- Scope keeps expanding
